Detection Engineering Interview Preparation

Common interview questions, technical scenarios, and preparation strategies for detection engineering and security operations roles.

Technical Questions

Detection approach:

  • Email gateway: Suspicious sender domains, failed SPF/DKIM/DMARC, URL reputation
  • User behavior: Clicks on links from external senders, downloads from email
  • Endpoint: Office spawning PowerShell/cmd, macro execution, suspicious child processes
  • Network: Connections to newly registered domains, beaconing patterns
  • Post-compromise: Credential harvesting attempts, lateral movement

Example KQL:

EmailEvents
| where SenderFromDomain !endswith "company.com"
| where UrlCount > 0
| join EmailUrlInfo on NetworkMessageId
| where UrlDomain in (ThreatIntelligenceIndicator | where ThreatType == "Phishing" | project DomainName)

Strategies:

  • Baseline analysis: Understand normal behavior before alerting on anomalies
  • Whitelisting: Exclude known-good activity (admin accounts, scheduled tasks)
  • Threshold tuning: Adjust count/time thresholds based on environment
  • Context enrichment: Add user/asset context to filter benign activity
  • Multi-signal correlation: Require multiple indicators before alerting
  • Time-based filtering: Exclude maintenance windows, business hours considerations

Process: Monitor false positive rate, categorize FP reasons, implement targeted exclusions, document tuning rationale.

Attack: Request TGS tickets for service accounts to crack offline (T1558.003)

Detection indicators:

  • Event ID 4769 (Kerberos Service Ticket Request)
  • Encryption type 0x17 (RC4) - weaker, preferred by attackers
  • High volume from single user
  • Targeting service accounts (SPNs)

KQL:

SecurityEvent
| where EventID == 4769
| where TicketEncryptionType == "0x17"  // RC4
| where ServiceName !endswith "$"  // Not machine accounts
| summarize RequestCount = count(), 
            UniqueServices = dcount(ServiceName)
            by AccountName, IpAddress, bin(TimeGenerated, 5m)
| where RequestCount > 10 or UniqueServices > 5

Scenario Questions

Step 1: Alert triage

  • Who: User account, admin or standard?
  • What: Full command line, source URL
  • Where: Endpoint name, business unit
  • When: Time, during business hours?

Step 2: Context gathering

  • Parent process (how was PS launched?)
  • User's recent activity
  • Is this user/host known for admin scripts?

Step 3: IOC analysis

  • URL reputation (VirusTotal, URLhaus)
  • Downloaded file hash (if available)
  • Domain age and registration

Step 4: Scope assessment

  • Any other hosts connecting to same URL?
  • Did the file execute?
  • Post-download activity?

Step 5: Decision

  • Benign: Document, add exclusion if recurring legitimate activity
  • Malicious: Escalate, contain, investigate further

1. Research ransomware TTPs:

  • Initial access: Phishing, RDP brute force, exploits
  • Execution: PowerShell, WMI, scheduled tasks
  • Persistence: Services, registry, scheduled tasks
  • Defense evasion: Disable AV, delete backups
  • Impact: Mass file encryption, ransom note

2. Map to MITRE ATT&CK:

  • T1486 (Data Encrypted for Impact)
  • T1490 (Inhibit System Recovery)
  • T1562 (Impair Defenses)

3. Build detections:

  • Shadow copy deletion (vssadmin, wmic)
  • Backup deletion patterns
  • Mass file rename operations
  • Known ransomware file extensions
  • Disable Windows Defender commands

4. Test and validate:

  • Use Atomic Red Team tests
  • Purple team exercises
  • Validate against known samples (sandboxed)

Behavioral Questions

STAR Format:

  • Situation: What was the security gap or problem?
  • Task: What was your specific responsibility?
  • Action: What detection did you build? (Be technical)
  • Result: What was the measurable impact?

Example structure:

"We had no visibility into [attack type]. I researched the TTPs, built a detection using [data source] that looks for [indicators]. After tuning, it detected [X incidents] in the first month, including [specific example]. False positive rate is under [X%]."

Resources to mention:

  • Threat intel reports (Mandiant, CrowdStrike, Microsoft)
  • MITRE ATT&CK updates
  • Security blogs (SpecterOps, Red Canary, Splunk)
  • Twitter/X security community
  • Conferences (BSides, DEF CON, Blue Team Con)
  • Sigma rules repository
  • Vendor documentation and updates

Practical application: "When I see a new technique, I assess if we have visibility, test if existing detections catch it, and build new detections if needed."
