NIST CSF Overview

The NIST Cybersecurity Framework provides a common language for managing cybersecurity risk. CSF 2.0 (released 2024) includes six core functions that organize cybersecurity activities.

Detect Function (DE)

The Detect function is where detection engineering directly contributes to organizational cybersecurity. It ensures timely discovery of cybersecurity events.

Detect Categories

Category ID Description Detection Engineering Role
Continuous Monitoring DE.CM Monitor for anomalies, indicators of compromise SIEM rules, EDR alerts, network monitoring
Adverse Event Analysis DE.AE Analyze detected events to understand attack Alert correlation, threat hunting, investigation

Respond Function (RS)

Detection engineering supports the Respond function through alert enrichment, automated response actions, and incident analysis capabilities.

Respond Categories

Category ID Detection Engineering Contribution
Incident Management RS.MA Incident creation, severity classification, assignment
Incident Analysis RS.AN Alert correlation, timeline generation, root cause
Incident Response Reporting RS.CO Automated notifications, stakeholder alerts
Incident Mitigation RS.MI SOAR playbooks, automated containment

Implementation Tiers

NIST CSF uses tiers to describe the degree to which an organization's cybersecurity risk management practices exhibit the framework characteristics.

Tier Name Detection Maturity
Tier 1 Partial Ad-hoc detection, reactive only, minimal coverage
Tier 2 Risk Informed Some detection rules, basic SIEM, limited correlation
Tier 3 Repeatable Documented detections, coverage metrics, tuning process
Tier 4 Adaptive Threat-informed, continuous improvement, automated response

ğŸ’¡ Tier Assessment Questions for Detection

  • Are detections mapped to threat intelligence (MITRE ATT&CK)?
  • Are detection metrics tracked and reported?
  • Is there automated response capability (SOAR)?
  • Are detections tested regularly (purple team, atomic tests)?

Interview Preparation

Primary alignment: Detect (DE) and Respond (RS) functions

Detect function:

  • DE.CM: Continuous monitoring through SIEM rules, EDR alerts
  • DE.AE: Adverse event analysis through correlation and hunting

Respond function:

  • RS.AN: Incident analysis through enriched alerts
  • RS.MI: Mitigation through automated response playbooks

Metrics to demonstrate: Detection coverage percentage, MTTD, false positive rate, MITRE ATT&CK coverage.

Key changes from CSF 1.1 to 2.0:

  • Govern function: New sixth function emphasizing cybersecurity governance
  • Supply chain: Enhanced focus on supply chain risk management
  • Broader scope: Applicable to all organizations (not just critical infrastructure)
  • Implementation examples: More practical guidance for implementation
  • Profiles: Simplified profile development process
