Hypothesis-Driven Hunting

Effective hunts start with a clear hypothesis about what threats may exist in your environment.

Hypothesis Sources

Intelligence-Based

  • Threat reports (APT campaigns)
  • Industry-specific threats
  • New vulnerability exploitation
  • Shared IOCs/TTPs

Situational Awareness

  • Anomalies in existing alerts
  • Gaps in detection coverage
  • High-value asset concerns
  • Recent changes in environment

Framework-Based

  • MITRE ATT&CK techniques
  • Uncovered attack stages
  • Common attacker patterns
  • Red team findings

Hypothesis Template

Hunt Hypothesis Document 
HUNT HYPOTHESIS: [Name]
Date: [Date]
Hunter: [Name]

HYPOTHESIS STATEMENT
"Adversaries may be using [TECHNIQUE] to [OBJECTIVE] in our environment,
which would be visible in [DATA SOURCE] as [OBSERVABLE]."

Example:
"Adversaries may be using scheduled tasks for persistence in our environment,
which would be visible in Windows Event Logs as Event ID 4698 from
non-standard user accounts or unusual task locations."

MITRE ATT&CK MAPPING
- Tactic: [TA00XX - Name]
- Technique: [T1XXX - Name]
- Sub-technique: [T1XXX.XXX - Name]

INTELLIGENCE BASIS
- [Source/Report that inspired this hunt]
- [Relevance to our environment/industry]

DATA SOURCES REQUIRED
- [Data source 1]: [Specific fields needed]
- [Data source 2]: [Specific fields needed]

SUCCESS CRITERIA
- Finding malicious activity
- Validating no malicious activity exists
- Identifying detection gaps
- Improving detection coverage

SCOPE
- Time range: [Start] to [End]
- Assets: [All / Specific subset]
- Exclusions: [Known benign activity to filter]

Interview Preparation

Structured approach:

  1. Hypothesis: Start with threat intel, MITRE technique, or anomaly
  2. Data identification: What logs/telemetry needed?
  3. Query development: Build detection queries
  4. Analysis: Review results, filter false positives
  5. Validation: Confirm findings are malicious or benign
  6. Documentation: Record methodology and findings
  7. Automation: Convert successful hunts to detections

Key point: The goal is to find threats AND improve detection coverage.
