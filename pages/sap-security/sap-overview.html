What is SAP? Complete Overview

Understanding SAP from the ground up - the company, its products, how enterprises use it, and why it's critical infrastructure that security teams must protect.

Why This Matters for Detection Engineers

SAP systems process 77% of the world's transaction revenue and hold the most sensitive business data. As a detection engineer, you'll need to integrate SAP logs into your SIEM, create detections for SAP-specific threats, and understand how attackers target these systems.

SAP: The Company

SAP SE (Systems, Applications & Products in Data Processing) is a German multinational corporation founded in 1972 by five former IBM engineers: Dietmar Hopp, Claus Wellenreuther, Hasso Plattner, Klaus Tschira, and Hans-Werner Hector. Originally named "Systemanalyse Programmentwicklung" (System Analysis Program Development), they created the first commercial enterprise resource planning (ERP) software.

What Does SAP Actually Do?

At its core, SAP provides Enterprise Resource Planning (ERP) software - integrated systems that manage all aspects of a business's operations. Think of SAP as the central nervous system of a large organization, connecting and coordinating:

The Problem SAP Solves

Before ERP systems like SAP, large enterprises faced a nightmare of disconnected systems:

✗

Without SAP/ERP

  • Finance uses one system
  • HR uses a different system
  • Manufacturing uses another
  • Sales tracks in spreadsheets
  • Data is siloed and inconsistent
  • Manual data entry between systems
  • No real-time visibility
  • Reporting takes weeks
✓

With SAP/ERP

  • Single integrated system
  • All departments connected
  • Single source of truth
  • Real-time data updates
  • Automated workflows
  • Cross-functional visibility
  • Instant reporting
  • Regulatory compliance built-in

Real-World Example: Order-to-Cash Process

Consider what happens when a customer places an order at a large manufacturer:

  1. Sales & Distribution (SD): Sales rep enters customer order in SAP
  2. Materials Management (MM): SAP checks inventory availability automatically
  3. Production Planning (PP): If inventory is low, SAP triggers production order
  4. Finance (FI): SAP creates accounting entries, checks customer credit
  5. Warehouse Management (WM): SAP generates picking list for warehouse
  6. Shipping: SAP creates delivery documents, shipping labels
  7. Accounts Receivable: SAP generates invoice automatically
  8. Controlling (CO): Profit margins calculated in real-time

All of this happens within one integrated system, with full audit trails - which is why SAP logs are so valuable for security monitoring.

SAP's Product Portfolio

SAP offers a wide range of products, but the core ERP systems are what most enterprises run:

Core ERP Products

SAP R/3 (Legacy - 1992-2015)

The system that made SAP dominant. "R/3" stands for "Real-time, 3-tier architecture" (presentation, application, database layers). While officially replaced by S/4HANA, many enterprises still run R/3 systems due to the complexity of migration. These legacy systems are often the most vulnerable and require careful security monitoring.

  • Architecture: Client-server, 3-tier
  • Database: Oracle, SQL Server, DB2, MaxDB
  • Status: Mainstream maintenance ends 2027
SAP ECC (ERP Central Component - 2004-Present)

Evolution of R/3, also known as "SAP ERP 6.0". This is what most enterprises currently run as their primary SAP system. ECC introduced Enhancement Packages (EhP) for incremental upgrades without full system replacement.

  • Architecture: NetWeaver-based
  • Database: Any major RDBMS
  • Current Status: Mainstream support until 2027, extended to 2030
  • Security Note: Most SAP security incidents target ECC systems
SAP S/4HANA (2015-Present) - Current Generation

SAP's current flagship ERP, built on the in-memory HANA database. "S/4" stands for "Simple Suite, 4th generation". This represents a complete re-architecture of SAP ERP for modern requirements including real-time analytics, simplified data models, and cloud-native deployment.

  • Database: SAP HANA only (in-memory)
  • Deployment: On-premise, Cloud (public/private), Hybrid
  • Interface: SAP Fiori (modern UI) + traditional SAP GUI
  • Key Advantage: Real-time analytics, simplified data model
Migration Reality

SAP has set 2027 as the deadline for ECC support, forcing enterprises to migrate to S/4HANA. This migration creates significant security risks and opportunities for detection engineers to ensure proper security controls during transition.

Industry-Specific Use Cases

Manufacturing Industry

Manufacturing companies use SAP to manage:

  • Production Planning (PP): Manufacturing schedules, capacity planning
  • Quality Management (QM): Inspection lots, quality certificates
  • Plant Maintenance (PM): Equipment maintenance, work orders
  • Materials Management (MM): Inventory, procurement, vendor management

Security Focus: Industrial espionage, production disruption, supply chain attacks, theft of BOMs (Bill of Materials)

Retail Industry

Retailers use SAP to manage:

  • Sales & Distribution (SD): Orders, pricing, promotions
  • Retail (IS-Retail): Store operations, POS integration
  • Supply Chain: Demand planning, replenishment
  • Customer Management: Loyalty programs, customer data

Security Focus: Customer PII theft, payment card data, pricing manipulation, inventory theft

Financial Services

Banks and financial institutions use SAP for:

  • Banking (IS-Banking): Core banking operations
  • Treasury: Cash management, financial instruments
  • Risk Management: Credit risk, market risk
  • Regulatory Reporting: Basel III, IFRS compliance

Security Focus: Financial fraud, regulatory violations, data breaches, insider trading

Healthcare Industry

Healthcare organizations use SAP for:

  • Healthcare (IS-H): Patient administration, billing
  • Supply Chain: Medical supplies, pharmaceuticals
  • HR: Staff scheduling, credentials management
  • Finance: Revenue cycle management

Security Focus: PHI/HIPAA compliance, medical identity theft, drug diversion, ransomware

Why SAP Systems Are High-Value Targets

Critical Security Consideration

SAP systems contain the "crown jewels" of an organization - financial records, customer data, intellectual property, employee information, and trade secrets. A breach of SAP can be more damaging than any other system in the enterprise.

What Attackers Want from SAP

Target Data SAP Module Attack Motivation
Financial Records FI/CO Fraud, insider trading, competitive intelligence
Customer Data SD, CRM Identity theft, phishing campaigns, data breach ransom
Employee Data HR/HCM Identity theft, social engineering, tax fraud
Vendor/Payment Data MM, AP Payment fraud, vendor impersonation
Product Designs/BOMs PP, PLM Industrial espionage, counterfeiting
Pricing Information SD Competitive advantage, market manipulation
M&A Information FI/CO Insider trading, competitive intelligence

Common SAP Attack Vectors

  1. Credential Theft: Phishing for SAP credentials, password spraying against SAP systems
  2. Privilege Escalation: Exploiting overly permissive SAP roles to gain SAP_ALL or similar privileges
  3. RFC Exploitation: Remote Function Calls can be abused to execute code or extract data
  4. Transport Manipulation: Injecting malicious code through the transport system
  5. Debug/Replace: Using debug privileges to bypass controls and modify data in memory
  6. Direct Table Access: Using SE16/SE16N to directly read or modify database tables
  7. Gateway Attacks: Exploiting SAP Gateway for unauthorized access
  8. Custom Code Vulnerabilities: ABAP code with SQL injection, hardcoded credentials

Interview Preparation: SAP Overview Questions

Common Interview Questions

Q: What is SAP and why is it important for security?

Answer: SAP is an Enterprise Resource Planning (ERP) system that serves as the central nervous system for large organizations, managing everything from finance and HR to supply chain and manufacturing. From a security perspective, SAP is critical because it contains the most sensitive business data - financial records, customer information, employee PII, trade secrets, and intellectual property. 77% of world transaction revenue touches a SAP system, making it a high-value target. As a detection engineer, monitoring SAP is essential because a breach can lead to financial fraud, data theft, regulatory violations, and severe business disruption.

Q: What's the difference between SAP ECC and S/4HANA?

Answer: SAP ECC (ERP Central Component) is the previous generation of SAP ERP, running on traditional databases like Oracle or SQL Server. S/4HANA is SAP's current-generation ERP, built exclusively on the in-memory HANA database. S/4HANA offers real-time analytics, a simplified data model, and modern user interfaces through SAP Fiori. From a security perspective, both systems need monitoring, but the logging and integration approaches differ. Many organizations are currently migrating from ECC to S/4HANA (deadline 2027), which creates additional security risks during transition that detection engineers should monitor.

Q: What kind of data does SAP contain that attackers target?

Answer: SAP contains multiple categories of high-value data: (1) Financial data - general ledger, accounts payable/receivable, which enables financial fraud; (2) Customer data - names, addresses, payment information for identity theft; (3) Employee data - SSNs, salaries, personal details; (4) Vendor/supplier data - bank accounts for payment fraud; (5) Intellectual property - product designs, pricing strategies, M&A information; (6) Operational data - production schedules, inventory levels for competitive intelligence. The combination of all this data in one system makes SAP a single point of compromise for an entire organization's most sensitive information.
